Issue Tracker

The AnguHashBlog's Issue Tracker serves as a vital tool for collaborators within our open-source community, providing a centralized overview of feature implementation across different application versions.

As explained in multiple places, when an issue exists in one app version, the same implementation should be applied across all app versions until a better implementation is accepted for global use across all apps.

From here collaborators can easily navigate to an existing implementation's PR, after choosing an issue to work on, and reuse the code from the other app version.

Some links of features that have not been implemented will not lead to a specific issue but rather to the issues tab on the relevant app. This means that the issue has not yet been documented for that app version. Collaborators need to document the issue before starting to work on it.

Always double-check if an implementation exists BEFORE starting to work on an issue since information is manually updated and can occasionally be outdated.

Always check the most current code since sometimes adjustments were made after the PR was submitted.
